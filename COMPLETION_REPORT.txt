1. WHY JAVAFX OVER SWING?
   - Modern UI framework (released after Swing)
   - FXML allows separation of UI and logic (like HTML/JS)
   - Better styling with CSS
   - More powerful animations and effects
   - Scene graph architecture
   - Better support for modern UIs

2. HOW DOES THE ALARM SYSTEM WORK?
   - javafx.animation.Timeline runs in background
   - Checks every 1 second (KeyFrame with 1s duration)
   - Compares current LocalDateTime with reminder times
   - When match found, creates modal Stage (popup)
   - Plays system beep using java.awt.Toolkit.beep()
   - User chooses TAKEN/MISSED/STOP
   - Action saved to database with timestamp
   - Guardians notified automatically

3. HOW DO GUARDIAN NOTIFICATIONS WORK?
   - When patient clicks TAKEN/MISSED
   - AlarmService calls userController.notifyGuardians()
   - Gets all GuardianPatientLink for that patient
   - Creates Notification object for each guardian
   - Saves to notifications table
   - Guardian sees unread badge (red circle with count)
   - Guardian can view notification details
   - Mark as read changes color from yellow to gray

4. EXPLAIN THE DATABASE SCHEMA
   - users: Patient & Guardian accounts (role column)
   - medicines: Medicine records
   - reminders: Scheduled reminders (status: PENDING/TAKEN/MISSED)
   - dose_history: Records of patient actions
   - guardian_patient_links: Many-to-many relationship
   - notifications: Guardian notifications
   - inventory: Stock tracking (future use)

5. WHAT IS MVC PATTERN?
   - Model: Data objects (Medicine, Reminder, User)
   - View: UI components (FXML files)
   - Controller: Logic between Model and View
   - Benefits: Separation of concerns, easier testing, maintainability

6. HOW IS AUTHENTICATION HANDLED?
   - Password hashed using SHA-256 (MessageDigest)
   - Hash stored in database (not plain text)
   - Login compares hashed passwords
   - Role verified (Patient vs Guardian)
   - Session managed with currentUser object

7. EXPLAIN COLOR-CODED STATUS
   - TableCell factory overrides updateItem() method
   - Checks status value (TAKEN/MISSED/PENDING)
   - Sets text style with -fx-text-fill property
   - Green (#2ecc71) for TAKEN
   - Red (#e74c3c) for MISSED
   - Orange (#f39c12) for PENDING
